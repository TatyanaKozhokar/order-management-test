# order-management-tests

Функциональные автотесты веб-приложения **«Портал управления заказами»** (ПУЗ) —
внутренней системы компании для обработки B2B-заявок от корпоративных клиентов.

---

## О системе

ПУЗ автоматизирует полный цикл работы с заказами: от приёма заявки до передачи
в службу доставки. Система используется тремя группами сотрудников:

| Роль | Задачи в системе |
|---|---|
| **Менеджер по продажам** | Создаёт заказы, заполняет данные клиента и оплаты |
| **Оператор склада** | Подтверждает или отклоняет заказы, проверяет наличие товара |
| **Администратор** | Управляет справочниками пользователей и продуктов, выгружает отчёты |

---

## Модули системы

### Заказы

Основной модуль. Поддерживает следующий жизненный цикл:

```
PENDING → APPROVED → SHIPPED → DELIVERED
    └──────────────→ CANCELLED
```

**PENDING** — заказ создан, ожидает подтверждения оператором.
**APPROVED** — оператор подтвердил наличие товара, заказ передан в доставку.
**CANCELLED** — заказ отменён на любом этапе до отгрузки.
**SHIPPED / DELIVERED** — статусы, проставляемые службой доставки через API.

Ключевые операции:
- Создание заказа с указанием клиента, товара, количества и способа оплаты
- Поиск и фильтрация по клиенту, статусу, дате и сумме
- Массовое подтверждение и отмена
- Экспорт реестра заказов в Excel (.xlsx)

### Пользователи

Справочник сотрудников и корпоративных клиентов. Роли: `ADMIN`, `MANAGER`, `OPERATOR`, `CUSTOMER`.
Управление доступно только администратору.

### Продукты

Каталог товаров с ценами. Продукты привязаны к заказам; при изменении цены товара
стоимость существующих заказов не пересчитывается (цена фиксируется на момент создания).

## Тестовое окружение

**URL:** `https://orders.internal.example.com`
**Тестовый стенд:** изолирован от production, данные сбрасываются ежесуточно в 03:00.

Учётные данные для запуска тестов хранятся в `src/test/resources/application-test.properties`.
Не коммить реальные пароли — используй переменные окружения:

```
TEST_ADMIN_LOGIN=...
TEST_ADMIN_PASSWORD=...
```

| Отмена заказа | `OrderManagementTest` |
| Поиск по имени клиента | `OrderManagementTest` |
| Экспорт реестра в Excel | `OrderManagementTest` |

## Запуск

```bash
# Все тесты
./gradlew test

# Конкретный класс
./gradlew test --tests "OrderManagementTest"

# С явными учётными данными
TEST_ADMIN_LOGIN=admin TEST_ADMIN_PASSWORD=secret ./gradlew test
```

Отчёт после прогона: `build/reports/tests/test/index.html`
